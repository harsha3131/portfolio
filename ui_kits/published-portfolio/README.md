# UI Kit — Published Portfolio

The generated, public-facing portfolio site that the product outputs. A single-page,
client-routed experience: **Home → Work → Project detail → Contact**, all over soft
photographic backgrounds with frosted-glass panels.

## Run
Open `index.html`. It loads the compiled design-system bundle (`_ds_bundle.js`) plus the
screen files below, in order. The bundle is generated by the compiler — if the page is
blank, the bundle hasn't compiled yet.

## Files
| File | Role |
|------|------|
| `index.html` | Mount point; loads bundle, React, Babel, then the screens. |
| `data.jsx` | Demo content — photo URLs (`SH_PHOTOS`) and projects (`SH_PROJECTS`). |
| `PhotoBg.jsx` | Shared photographic background (light wash + slow drift) + `Eyebrow`. |
| `Hero.jsx` | Landing view — avatar, availability badge, gradient headline, CTAs. |
| `Work.jsx` | Filterable project gallery in `GlassCard`s. |
| `ProjectDetail.jsx` | Case-study view with meta cards + next-project link. |
| `Contact.jsx` | Closing CTA with a glass contact form. |
| `App.jsx` | Floating `GlassNav`, simple route state, footer. |

## Composition
Screens compose primitives from the system (`GlassNav`, `GlassCard`, `Button`, `Badge`,
`Tag`, `Avatar`, `Input`) — they do **not** re-implement them. Each screen attaches itself
to `window` so the next Babel script can use it (Babel scripts don't share scope).

## Backgrounds
Photographs are loaded from Unsplash CDN URLs in `data.jsx`. Swap them for the user's own
photography — the glass + wash treatment keeps text legible over anything.
